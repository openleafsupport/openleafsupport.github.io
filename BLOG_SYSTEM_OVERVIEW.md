# Blog System Overview

This document provides a high-level overview of the complete blog publishing system.

## System Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                    YOUR SOURCE FOLDER                            │
│         _published-articles/ (nested YYYY/MM/DD-title.md)         │
│              + images in same folder (image-1.jpg)                │
│                                                                   │
│           _data/featured_articles.json (rankings 1-6)            │
│                                                                   │
│              ↓ ruby scripts/build_posts.rb                        │
│              ↓ ruby scripts/sync_featured_articles.rb             │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                    JEKYLL PUBLISH FOLDER                         │
│            _posts/ (flat YYYY-MM-DD-title.md)                    │
│         _posts/YYYY/MM/ (images copied from source)              │
│                                                                   │
│              ↓ bash scripts/validate_blog.sh                      │
│              ↓ bundle exec jekyll build                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   PRODUCTION LOCATION                            │
│              assets/post-assets/YYYY/MM/                         │
│               (images synced by validate script)                  │
│                                                                   │
│                 _site/ (built HTML/CSS/JS)                       │
│         (generated by Jekyll, deployed to GitHub Pages)          │
└─────────────────────────────────────────────────────────────────┘
```

## Key Folders & Files

### Source: `_published-articles/`
Your working folder where you create and edit posts and images.

```
_published-articles/
  ├── 2025/
  │   ├── 07/
  │   │   ├── 11-Why-Reading-Matters.md
  │   │   └── image-1.png
  │   └── 12/
  │       ├── 31-How-I-Encouraged.md
  │       ├── image-1.jpeg
  │       └── image-2.jpg
  └── 2026/
      └── ...
```

**Files:** Post markdown + images (simple filenames like `image-1.jpeg`)

### Configuration: `_data/featured_articles.json`
Centralized management of featured articles with rankings.

```json
{
  "featured": [
    {"rank": 1, "file": "_published-articles/2025/07/11-Why-Reading-Matters.md"},
    {"rank": 2, "file": "_published-articles/2026/01/01-How-to-Build.md"},
    ...
  ]
}
```

**Validation Rules:**
- Ranks must be 1-6
- No duplicate ranks
- All file paths must exist

### Published: `_posts/`
Generated folder that Jekyll reads from. Files here are auto-generated.

```
_posts/
  ├── 2025-07-11-Why-Reading-Matters.md
  ├── 2025-12-31-How-I-Encouraged.md
  ├── 2025/
  │   ├── 07/image-1.png
  │   └── 12/image-1.jpeg
  └── ...
```

**Files:** Auto-generated markdown files + images (copied by build script)

### Production: `assets/post-assets/`
Final image location served by the website.

```
assets/post-assets/
  ├── 2025/
  │   ├── 07/image-1.png
  │   └── 12/
  │       ├── image-1.jpeg
  │       └── image-2.jpg
  └── ...
```

**Files:** Images synced from `_posts/YYYY/MM/` by validation script

## Complete Workflow

### 1. Create a New Post

Create folder structure:
```bash
mkdir -p _published-articles/2025/07
```

Create markdown file with simple image references:
```markdown
# _published-articles/2025/07/11-My-Post.md
---
layout: post
title: "My Post Title"
description: "Short description"
categories:
  - Reflections
cover_image: "image-1.jpeg"  # Just the filename
---

Here's some text with an image:
![Alt text](image-2.jpg)  # Just the filename
```

Add images to the same folder:
```
_published-articles/2025/07/image-1.jpeg
_published-articles/2025/07/image-2.jpg
```

### 2. Manage Featured Articles

Edit `_data/featured_articles.json` to add/reorder featured articles:
```json
{
  "rank": 4,
  "file": "_published-articles/2025/07/11-My-Post.md"
}
```

### 3. Build & Sync

Sync featured values to markdown:
```bash
ruby scripts/sync_featured_articles.rb
```

Build posts and images:
```bash
ruby scripts/build_posts.rb
```

Results:
- Markdown → `_posts/2025-07-11-My-Post.md` (flat naming for Jekyll)
- Images → `_posts/2025/07/image-*.jpeg` (copied to maintain structure)
- Featured field → automatically added/updated in markdown

### 4. Validate

Run validation:
```bash
bash scripts/validate_blog.sh
```

Checks:
- ✅ Post structure (frontmatter, categories, content)
- ✅ Image references exist
- ✅ Featured configuration (ranks 1-6, no duplicates)
- ✅ Image standards (size, dimensions)
- ✅ Syncs images to `assets/post-assets/`

### 5. Build Jekyll

```bash
bundle exec jekyll build
```

Jekyll:
- Reads posts from `_posts/YYYY-MM-DD-title.md`
- Uses Jekyll plugin to convert: `![Alt](image-1.jpeg)` → `/assets/post-assets/2025/07/image-1.jpeg`
- Generates HTML to `_site/`

### 6. Commit & Deploy

```bash
git add _published-articles/ _posts/ _data/ assets/
git commit -m "Add post: My Post Title"
git push
```

GitHub Actions CI/CD:
1. Checks out repo
2. Runs `jekyll build`
3. Uploads `_site/` to GitHub Pages
4. Website is live!

## Scripts Overview

### `scripts/build_posts.rb`
- **Purpose**: Generate posts and copy images from source to Jekyll folder
- **Input**: `_published-articles/YYYY/MM/DD-title.md` + images
- **Output**: `_posts/YYYY-MM-DD-title.md` + images in `_posts/YYYY/MM/`
- **Run**: `ruby scripts/build_posts.rb`

### `scripts/sync_featured_articles.rb`
- **Purpose**: Sync featured article rankings from JSON to markdown
- **Input**: `_data/featured_articles.json`
- **Output**: `featured: X` added/updated/removed in markdown frontmatter
- **Run**: `ruby scripts/sync_featured_articles.rb`

### `scripts/validate_blog.sh`
- **Purpose**: Validate posts, images, and configuration
- **Checks**: Structure, references, featured config, image standards
- **Output**: Images synced to `assets/post-assets/`
- **Run**: `bash scripts/validate_blog.sh`

## File Reference

| File | Location | Purpose |
|------|----------|---------|
| Post markdown | `_published-articles/YYYY/MM/DD-title.md` | Source post content |
| Post images | `_published-articles/YYYY/MM/image-*.ext` | Source images |
| Featured config | `_data/featured_articles.json` | Featured article rankings |
| Generated posts | `_posts/YYYY-MM-DD-title.md` | Posts for Jekyll |

| Built site | `_site/` | Final HTML (deployed) |

## Tips & Best Practices

1. **Always edit source** — Edit in `_published-articles/`, not `_posts/`
2. **Use JSON for featured** — Never manually edit `featured:` in markdown
3. **Validate before building** — Catch errors early
4. **Test locally** — Use `jekyll serve` to preview before deploying
5. **Commit source + generated** — Both `_published-articles/` and `_posts/` should be in git
6. **Use simple image names** — `image-1.jpeg`, `cover.png` (no full paths in markdown)
7. **Keep images in post folder** — Images live with the markdown source

## Troubleshooting

**Images showing 404?**
1. Check image is in `_published-articles/YYYY/MM/`
2. Run `ruby scripts/build_posts.rb` to copy images
3. Run `bash scripts/validate_blog.sh` to sync to assets
4. Rebuild with `bundle exec jekyll build`

**Featured not showing?**
1. Check `_data/featured_articles.json` has correct path
2. Run `ruby scripts/sync_featured_articles.rb`
3. Run `bash scripts/validate_blog.sh` to validate
4. Check markdown has `featured: X` in frontmatter

**Validation errors?**
1. Run `bash scripts/validate_blog.sh` to see details
2. Fix the issue (missing files, invalid ranks, etc.)
3. Re-run validation to confirm fix
