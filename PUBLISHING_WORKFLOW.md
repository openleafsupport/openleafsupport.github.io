# Publishing Workflow

This blog uses a two-stage publishing system to keep source posts organized and version-controlled.

## Structure

```
_published-articles/          # Source posts (nested structure)
  ├── 2025/
  │   ├── 07/
  │   │   └── 11-Why-Reading-Matters...md
  │   └── 12/
  │       └── 31-How-I-Encouraged...md
  └── 2026/
      └── ...

_posts/                       # Generated posts (flat structure for Jekyll)
  ├── 2025-07-11-Why-Reading-Matters...md
  ├── 2025-12-31-How-I-Encouraged...md
  └── ...

assets/post-assets/           # Post images (nested by year/month)
  └── 2025/12/
      ├── image-1.jpeg
      └── image-2.jpg
```

## Creating a New Post

1. **Create the post file** in `_published-articles/`:
   ```
   _published-articles/YYYY/MM/DD-post-title.md
   ```

   Example: `_published-articles/2025/12/25-Christmas-Reading-List.md`

2. **Add post content** (frontmatter + markdown):
   ```yaml
   ---
   layout: post
   title: "Your Post Title"
   description: "Short description"
   categories:
     - Reflections  # or: book-reviews, updates, announcements
   tags:
     - tag1
     - tag2
   cover_image: "image-2.jpg"  # Just the filename (goes in assets/post-assets/YYYY/MM/)
   featured: 1  # Optional: set to 1-6 to feature on homepage
   ---
   
   # Your post content
   
   ![Alt text](image-1.jpeg)  # Just the filename
   ```

3. **Add images** to `assets/post-assets/YYYY/MM/`:
   - Name them: `image-1.jpeg`, `image-2.jpg`, etc.
   - They're referenced in markdown as: `![Alt](image-1.jpeg)` (auto-resolved to full paths by Jekyll plugin)

4. **Build and publish**:
   ```bash
   ruby scripts/build_posts.rb   # Generates posts to _posts/
   bundle exec jekyll build       # Builds the site
   ```

5. **Commit both**:
   ```bash
   git add _published-articles/ _posts/ assets/
   git commit -m "Add new post: Christmas Reading List"
   git push
   ```

## Workflow Summary

| Step | Location | Action |
|------|----------|--------|
| **1. Write** | `_published-articles/YYYY/MM/DD-title.md` | Create and edit source post |
| **2. Add Images** | `assets/post-assets/YYYY/MM/` | Store images (referenced as `image-1.jpeg` in markdown) |
| **3. Build** | Run `ruby scripts/build_posts.rb` | Generates flat-structure posts to `_posts/` |
| **4. Preview** | Run `bundle exec jekyll serve` | Test locally at `http://localhost:4000` |
| **5. Commit** | Git commit | Version control both `_published-articles/` and `_posts/` |
| **6. Deploy** | GitHub Actions | CI/CD automatically builds and deploys |

## Key Points

- **Source of truth**: `_published-articles/` — edit here
- **Published form**: `_posts/` — generated by build script, committed to git
- **Jekyll reads from**: `_posts/` only (for static site generation)
- **Image references**: Use simple filenames like `image-1.jpeg` — Jekyll plugin auto-resolves to `/assets/post-assets/YYYY/MM/image-1.jpeg`
- **Cover images**: Set `cover_image: "image-2.jpg"` in frontmatter — automatically resolved to correct path

## Updating an Existing Post

1. Edit the file in `_published-articles/YYYY/MM/`
2. Run `ruby scripts/build_posts.rb` (copies updated version to `_posts/`)
3. Commit both
4. Push to deploy

## Build Script Details

The `scripts/build_posts.rb` script:
- Scans `_published-articles/` for posts with nested structure (YYYY/MM/DD-title.md)
- Converts to flat naming (YYYY-MM-DD-title.md) for Jekyll
- Copies posts to `_posts/` (doesn't delete originals)
- Can be run multiple times safely
